03Design Decisions

The six questions every system like this has to answer. Click each to see the tradeoff and our pick.

Key hashing — Argon2id vs HMACHMAC-SHA256 + pepper
Tradeoff

Argon2id is slow and memory-hard — perfect for passwords, where you need to make brute force expensive. But API keys already carry 32 bytes of entropy, so brute force is infeasible regardless of hash speed. At 5k rps, Argon2id (~100ms per hash) would melt the CPU.

Pick — HMAC-SHA256 with a server-side pepper

Microsecond-fast, deterministic (so we can index key_hash in Postgres), and the pepper means a database leak alone is not enough for an attacker to verify candidate keys.

Rate limit algorithmToken bucket via Redis Lua
Options

Fixed window is simple but has a 2× burst at window boundaries. Sliding window log is precise but memory-heavy. Token bucket allows controlled bursts — which is what real APIs need.

Pick — Token bucket via Redis Lua script

Industry standard (AWS, Stripe, GitHub). Tiny state per key: {tokens, last_refill_ts}. Lua makes the check + decrement atomic in one round-trip.

Usage pipelineRedis INCR + batch flush
The math

5,000 rps × 86,400 = ~430M events/day. Writing every event to Postgres = 5k writes/sec sustained. Postgres could do it, but it is wasteful.

Pick — Redis INCR on the hot path, background flusher every 30–60s

Hot path adds one Redis op (essentially free). Flusher reads usage:{tenant}:{key}:{minute} keys and writes one row per minute into Postgres. Drops write rate from 5k/sec to ~10/sec — a 500× reduction. Idempotent via INSERT ... ON CONFLICT.

Rate limit vs daily quotaTwo mechanisms
Why separate

Rate limit = burst protection (e.g. 100 req/sec, rolling). Quota = billing & fairness (e.g. 1M req/month, calendar-aligned). Different time scales, different semantics, different consistency requirements.

Pick — Token bucket for rate limit, simple counter for quota

Both checks happen in the same Lua script, so it is still one Redis round-trip on the hot path.

p99 latency budgetp50 10ms · p99 20ms
The breakdown

Network in (1–2ms) + HMAC (<1ms) + Redis Lua (2–4ms) + response (1–2ms) = ~6–8ms p50, ~15ms p99. 20ms p99 is comfortable.

Critical optimization

Pipeline all Redis ops into a single Lua call. Cache lookup, rate check, quota check, usage increment in one round-trip. This is the difference between hitting and missing the budget.

Rotation grace period48h overlap + force
Tradeoff

Tenants need time to update deployments after rotating a key. Immediate cutoff breaks them. Infinite grace makes rotation meaningless.

Pick — 48-hour overlap by default, force=true for immediate revoke

Normal rotation gives a calm migration window. force=true handles the leaked-key compromise case.
